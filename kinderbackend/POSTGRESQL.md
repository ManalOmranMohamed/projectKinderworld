# PostgreSQL Readiness

The backend can now run against PostgreSQL through `DATABASE_URL`.

## Recommended connection URL

```bash
DATABASE_URL=postgresql+psycopg://postgres:postgres@localhost:5432/kinderbackend
```

The app also accepts legacy `postgres://...` URLs and normalizes them automatically.

## Driver

Install:

```bash
pip install "psycopg[binary]"
```

## First-time setup

```bash
cd kinderbackend
python -m alembic upgrade head
```

## Optional pool tuning

```bash
DB_POOL_SIZE=5
DB_MAX_OVERFLOW=10
DB_POOL_RECYCLE_SECONDS=1800
```

## SQLite fallback

If `DATABASE_URL` is not set, the backend still falls back to local SQLite for development only:

```text
sqlite:///kinder.db
```

## Remaining migration guidance

- Prefer PostgreSQL in staging/production.
- Review autogenerated Alembic migrations before applying them.
- SQLite test-only assumptions should not be copied into production configuration.
